# Contract: Autosync Markers

## Marker syntax

HTML comments delimit regenerable regions:

```markdown
<!-- AUTOSYNC:BEGIN:<section-name> -->
...content regenerated by scripts/sync-readme.sh...
<!-- AUTOSYNC:END -->
```

Valid `<section-name>` values: `agent-count`, `agent-table`, `scope-summary`, `pipeline-diagram`.

## 4 Autosync regions

### Region 1: `agent-count`

**Location**: README intro paragraph.

**Source**: count of non-separator rows in SKILL.md Agent Reference table.

**Generated content** (example):

```markdown
<!-- AUTOSYNC:BEGIN:agent-count -->
The centerpiece is the **Task** skill — a scope-adaptive orchestrator that runs up to 15 specialized agents through a complete development lifecycle.
<!-- AUTOSYNC:END -->
```

Only the sentence is regenerated; surrounding context preserved.

### Region 2: `agent-table`

**Location**: `## Agents` section in README.

**Source**: `skills/task/SKILL.md` Agent Reference table + `skills/task/agents/refs/model-tiers.md` for tiers.

**Generated content**: markdown table with columns `#`, `Agent`, `Model`, `What It Does`, `Strict-tier approval`.

### Region 3: `scope-summary`

**Location**: `### Scope-Adaptive Pipeline` subsection in README.

**Source**: `skills/task/agents/refs/scope-pipelines.md` scope-family summary.

**Generated content**: 5-row table (XS/S/M/L/XL → pipeline family).

### Region 4: `pipeline-diagram`

**Location**: `### Pipeline` section in README.

**Source**: `skills/task/agents/refs/pipelines.md` Pipeline Overview ASCII.

**Generated content**: the ASCII art block as-is (copied).

## Generator rules

- Each region has a named bash function in `scripts/sync-readme.sh`: `generate_agent_count()`, `generate_agent_table()`, `generate_scope_summary()`, `generate_pipeline_diagram()`.
- Each function reads its source files (via `grep`/`awk`/`sed` or `jq`), produces markdown, and returns on stdout.
- The main `sync` function reads README, finds each AUTOSYNC region, replaces its content with the generator output.

## Idempotency guarantee

Running `sync-readme.sh` twice in a row with no source changes MUST produce zero byte diff between runs. (SC-008 verification.)

Implemented by:
- Deterministic extraction (no timestamps, no random order).
- Stable sort where order matters.
- Atomic write (tmp file + mv).

## Initial marker insertion

If README does not contain AUTOSYNC markers (first-ever sync run):

1. Script attempts to locate canonical sections by H2/H3 heading pattern match.
2. If all 4 regions found: wrap each in markers, write back, exit 0.
3. If any region can't be located: exit 3 with instructions showing the expected headings.

After first successful run, subsequent runs find existing markers and only replace content between them.

## Exit codes

| Code | Meaning |
|---|---|
| 0 | Success — README updated or already in sync |
| 1 | Parse error in source file |
| 2 | Source file missing |
| 3 | Cannot locate section for initial marker insertion |

## Hook behavior

`scripts/install-hooks.sh` writes `.git/hooks/pre-commit` that:

1. Runs `scripts/sync-readme.sh`.
2. If README was modified: `git add README.md` + continue commit.
3. If sync exits non-zero: abort commit with error.

User opt-in via `./scripts/install-hooks.sh`. Never installed automatically.
